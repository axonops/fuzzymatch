# Phase 8.5 Plan 15a: Q14b Mechanical Refactors + Gap 1 Rename Summary

Mechanical half of the M5-split former-Plan-15 — Q14b's `WriteGoldenFile` unexport with test-only re-export, the `var _ = func() bool {...}()` → explicit `func init()` refactor across all 23 dispatch_*.go files, and the Gap 1 mechanical rename of `scorer_options_test.go` to `scorer_options_internal_test.go`. No behavioural changes; full test suite green throughout.

## What Was Built

### Task 1 — WriteGoldenFile unexport + 23 dispatch init() refactor (commit d510bf4)

**WriteGoldenFile unexport (Q14b):**
- `golden_canonical.go` — `func WriteGoldenFile(path string, v any) error` renamed to `func writeGoldenFile(...)`. Wrapper-prefix string updated from `"fuzzymatch: WriteGoldenFile:"` to `"fuzzymatch: writeGoldenFile:"`. Godoc updated to point to the test-only re-export.
- `export_test.go` — added `var WriteGoldenFile = writeGoldenFile` so the five existing call sites (algorithms_golden_test.go:92, golden_canonical_test.go:240/263/284, golden_test.go:74/178) compile unchanged under the original capitalised name. The build-tag `_test.go` suffix excludes the re-export from production builds.
- `golden_canonical_test.go` — `TestWriteGoldenFile_WriteFailureSurfacesError` assertion updated to match the new `"fuzzymatch: writeGoldenFile:"` wrapper prefix.
- `ai_friendly_test.go` — `internalAllowlist` cleared (writeGoldenFile is no longer in the exported root-package AST surface; allowlist retained as an empty map for future use).

**23 dispatch_*.go init() refactor (Q14b option A):**
Per dispatch file, the package-level `var _ = func() bool { ... return true }()` registration block was replaced with an explicit `func init() { ... }` containing the same dispatch-table write (minus the `return true`). Each init's godoc cites the determinism-standards SKILL rationale: pure-write into a pre-allocated slot, no IO, no time, no goroutines, no ordering dependency on other init functions (T-08.5-28 mitigated by construction).
