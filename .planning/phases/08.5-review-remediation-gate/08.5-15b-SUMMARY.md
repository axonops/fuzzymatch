## Decisions Made

- **(SWGParams).Validate panic policy**: applied the Q2 strict-parameter framework — invalid construction is a programmer error and panics with a typed-error value wrapping `ErrInvalidSWGParam`. The shape mirrors `TverskyScore` and `MongeElkanScore` direct-call panic discipline (per docs/requirements.md §6.A).
- **Validate helper sharing**: a single private `validate() error` method drives BOTH the public `Validate()` panic-wrap AND the `NewSWGParams()` defence-in-depth self-test. This guarantees the two surfaces never diverge in their invariant definition.
- **runeAt rename**: dropped the never-used rune return value across 6 call sites. The new name `runeSizeAt(string, int) int` more accurately describes the function's role as a byte-size dispatch for phonetic-encoder skip paths.
- **gocyclo residue scope**: 16 of the 19 gocyclo violations are on canonical algorithm state machines (DoubleMetaphoneKeys CC=275, NYSIISCode CC=78, SoundexCode CC=29, MRACompare CC=23, MRACode CC=16, Validate CC=16, dmPrep CC=17, TverskyScore CC=13, TverskyScoreRunes CC=13, etc.). Each is annotated with `//nolint:gocyclo` and a one-line justification citing the originating primary source. None are refactoring candidates — extraction would obscure the recurrence / state-machine structure.
- **gosec G602 mra.go false positives**: 3 indexed-read warnings at the post-`step2Len > 6` early-return branch are documented false positives. gosec's bounds-prover does not propagate the negation. `//nolint:gosec` annotations include a one-line proof-sketch in the comment.
- **staticcheck SA1019 parser.ParseDir**: kept deliberately because `golang.org/x/tools` is outside the spec §1 zero-runtime-dep allowlist. The Plan 18 verification report's "v1.x revisit list" carries this as the sole residue requiring future re-evaluation.
