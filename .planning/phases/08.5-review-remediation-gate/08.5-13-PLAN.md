Public surface:
```go
// Warning carries a single diagnostic emitted by Validate.
type Warning struct {
    Algorithm AlgoID  // The algorithm the warning is scoped to (or AlgoIDNone for cross-cutting).
    Kind      WarnKind
    Detail    string  // Human-readable elaboration. Stable English; British spelling.
}

// WarnKind enumerates the diagnostic categories Validate produces.
type WarnKind int

const (
    WarnEmptyInput WarnKind = iota
    WarnUnequalLength
    WarnNoTokensAfterNormalise
    WarnAllNonASCIIDropped
    WarnPathologicallyLargeInput
)

func (k WarnKind) String() string

func Validate(a, b string) []Warning
```

Per-algorithm rule table (sketch — must match docs/requirements.md §11.5 "Per-WarnKind semantics"):
- WarnEmptyInput: ANY algorithm where one or both inputs are empty (typically returns 0.0 silently — flag as low-information)
- WarnUnequalLength: Hamming (silent-max behaviour; consumer may want to know)
- WarnNoTokensAfterNormalise: TokenSort, TokenSet, PartialRatio, TokenJaccard, MongeElkan (post-Tokenise empty token set)
- WarnAllNonASCIIDropped: Strcmp95, Soundex, NYSIIS, MRA (ASCII-only algorithms with non-ASCII input that drops to empty post-normalisation)
- WarnPathologicallyLargeInput: any algorithm with O(m·n) DP (DL-Full primarily) where len(a)*len(b) exceeds ~10^7 (consumer DoS-mitigation hint)

Determinism rules (per .claude/skills/determinism-standards/SKILL.md):
- No map iteration on Validate output paths.
- Warning slice sorted by (Algorithm, Kind) via sort.SliceStable with a complete sort key.
- Pure function; safe for concurrent use; never panics.

Pattern analogs:
- algoid.go:213-308 — AlgoID enum + String() — the mirror for WarnKind
- algoid.go:269-308 AlgoIDs() — the hand-written slice (NOT iterating a map) pattern; if WarnKinds() is added, follow this
- normalise.go + tokenise.go — pure-function pattern
