## Locked Discuss-Phase Decisions (Q1–Q14)

These decisions are the contract for Phase 8.5 implementation. They were made in the external 2026-05-17 session and are recorded across the four spec artefacts above.

### Q1 — Hamming public-API policy: silent-max
**Lock:** `HammingDistance(a, b string) int` and `HammingDistanceRunes(a, b string) int` return `max(len(a), len(b))` on unequal-length input. No error tuple. No `ErrUnequalLength` placeholder. `HammingScore` continues to return `0.0` on unequal length. **Code wins, spec catches up.**
**Sources:** `docs/requirements.md §7.1.4` (silent-max documented), `CHANGELOG.md "Breaking"` entry, `errors.go` header note about the unused placeholder will be removed.
**Reviewer reports closing:** algorithm-correctness Critical, code-reviewer Critical, bdd-scenario Critical, user-guide Critical, docs-writer Critical.

### Q2 — Data-vs-parameter validation framework
**Lock:** **Comparison-data inputs** (the two strings being compared) → lenient (no panic, no error, sensible value). **Parameter inputs** (`n`, `α`, `β`, `inner AlgoID`, `threshold`, `weight`, SWG params) → strict: typed error at Scorer-construction time, panic-with-sentinel on direct algorithm calls.
**Specific guards added:**
- `WithThreshold(t)`: `if math.IsNaN(t) || t < 0.0 || t > 1.0 { return ErrInvalidThreshold }`. `±Inf` already rejected by the range check.
- `WithAlgorithm(_, w)`: rejects NaN, ±Inf, w ≤ 0 → `ErrInvalidWeight`.
- `WithTverskyAlgorithm(_, α, β, _)`: rejects `α + β ≤ 0` → `ErrInvalidTverskyParam` (closes the panic-at-Score-time escape).
- Direct `TverskyScore` call panics with `ErrInvalidTverskyParam` on the same input.
**Sources:** `docs/requirements.md §6.A`, `CHANGELOG.md "Changed"` entries.
**Reviewer reports closing:** api-ergonomics Critical (×2), security Critical (×2), determinism Critical, test-analyst Critical (×2), code-reviewer Critical (×2), user-guide Critical, docs-writer Critical, go-quality Critical.

### Q3 — MongeElkan symmetric-by-default rename + inert opts removal
**Lock:** `MongeElkanScore` becomes the **symmetric** variant. The directional variant is renamed `MongeElkanScoreAsymmetric`. The `opts NormalisationOptions` parameter is removed from both (it had no effect inside Monge-Elkan).
**Breaking, pre-v1.0.** **Sources:** `docs/requirements.md §7.15`, `CHANGELOG.md "Breaking"`.
**Reviewer reports closing:** api-ergonomics Important (×3), code-reviewer Important.

### Q4 — Validate feature + sentinel hygiene + panic policy
**Lock:**
- New public surface: `fuzzymatch.Validate(a, b string) []Warning` + `Warning` struct + `WarnKind` enum + 5 constants (`WarnEmptyInput`, `WarnUnequalLength`, `WarnNoTokensAfterNormalise`, `WarnAllNonASCIIDropped`, `WarnPathologicallyLargeInput`). `WarnKind.String()` returns CamelCase.
- Three unused sentinels removed: `ErrInvalidConfiguration`, `ErrInvalidInput`, `ErrEmptyInput`. None had call sites.
- New sentinel `ErrInvalidInnerAlgo` added for the Monge-Elkan inner-AlgoID path (Q4 follow-up commit `c5b377d`). Monge-Elkan direct calls now panic with this sentinel.
- Every remaining sentinel carries the four-section godoc block (**What** / **Common causes** / **Resolution** / **Example**).
**Validate** must ship across the 6 required surfaces (README Quick Start / Common Patterns, `docs/algorithms.md` or `docs/best-practices.md`, per-algorithm godoc cross-refs, `llms.txt` + `llms-full.txt`, user-guide section, runnable `examples/` program).
**Sources:** `docs/requirements.md §6 + §6.A + §11.5`, `CHANGELOG.md "Added"`, `.planning/REQUIREMENTS.md` VALIDATE-01..VALIDATE-06.
**Reviewer reports closing:** docs-writer Critical (×3), bdd-scenario Critical (×3), test-writer Critical (×3), api-ergonomics Critical, user-guide Critical, security Important, test-analyst Important.

### Q5 — ASCII-only algorithms have no Runes variants; PartialRatioScoreRunes removed
**Lock:** Strcmp95, Soundex, Double Metaphone, NYSIIS, MRA, and the token-tier algorithms (Token Sort/Set/Partial/Jaccard, Monge-Elkan) ship no `*Runes` variants. `PartialRatioScoreRunes` is **removed** (breaking, pre-v1.0) — token-tier algorithms operate on the output of `Tokenise`, which is itself rune-aware; the byte-level Indel kernel is correct on post-`Tokenise` byte strings.
**Sources:** `docs/requirements.md §7.7, §7.16–§7.19, §7.20–§7.23`, `CHANGELOG.md "Breaking"`.
**Reviewer reports closing:** api-ergonomics Important (×3).

### Q6 — Ratcliff-Obershelp asymmetric-by-design (Q6a) + AlgoID.String / WarnKind.String CamelCase (Q6b)
**Q6a Lock:** Ratcliff-Obershelp is **asymmetric by design** for difflib parity. `PropAlgorithmScore_Symmetric` excludes `AlgoRatcliffObershelp`. The exception is documented in `.claude/skills/algorithm-correctness-standards/SKILL.md`. Re-confirms OQ-1 (locked 2026-05-14).
**Q6b Lock:** `AlgoID.String()` returns CamelCase ("Levenshtein", "JaroWinkler", "DamerauLevenshteinOSA", "NYSIIS", etc.). `WarnKind.String()` follows the same convention ("EmptyInput", "UnequalLength", etc.).
**Sources:** `docs/requirements.md §6 (table) + §7.24 + §11.5`, `CHANGELOG.md "Changed"`.
**Reviewer reports closing:** docs-writer Critical, api-ergonomics Important, algorithm-correctness Important.

### Q7 — DoubleMetaphone [4]byte + 5 performance-budget amendments + Long-input scope notes + Q-gram capacity hint
**Q7a Lock:** DoubleMetaphone primary/secondary key accumulators move from `strings.Builder` to `[dmMaxLen]byte` + length counters. Saves 4 allocs per `DoubleMetaphoneKeys` call. Returned strings unchanged.
**Q7b Lock:** §14.1 / §14.2 budgets revised to match implementation reality:
- DL-Full Medium: 8.2 µs (was < 3 µs) — Lowrance-Wagner structurally requires the full O(m·n) DP table.
- SWG Medium: < 10 µs (was < 5 µs).
- Normalise ASCII Short: ≤ 1 alloc (was 0) — the `string(buf)` return is structurally unavoidable without `unsafe.String`.
- Soundex Code: ≤ 1 alloc (was 0) — same `string(stackBuf[:n])` constraint.
- Scorer ASCII Short: ≤ 8 (was previously inconsistent); Scorer ASCII Medium: ≤ 20.
**Q7c Lock:** Long-input heap-fallback scope notes added to per-algorithm godoc where the optimisation is structurally absent.
**Q7d Lock:** `extractQGrams` adds 25 % capacity headroom (`(len(s)-n+1) + (len(s)-n+1)/4`) to reduce map-rehash probability at Medium.
**Sources:** `docs/requirements.md §14.1, §14.2, §14.3`.
**Reviewer reports closing:** algorithm-performance Critical (×4), Important (×7).

### Q8 — DL-Full Short ≤ 1 alloc + Tokenise ASCII fast path + Scorer §14.2 + Ratcliff-Obershelp ≤ 4 allocs
**Q8a Lock:** DL-Full Short ≤ 1 alloc (the stack-buffer optimisation that would achieve 0 allocs requires a 34 KB stack frame; judged too fragile).
**Q8b Lock:** `Tokenise` ASCII fast path: token boundaries become byte-index ranges; tokens returned as substrings of the input (`s[lo:hi]` zero-copy) when `opts.Lowercase == false`; a single scratch buffer for the lowercased case. Token-tier algorithms inherit the savings. Dedicated plan with BDD + benchmarks.
**Q8c Lock:** Scorer composite §14.2: Short ≤ 8 allocs (post-Q7a DoubleMetaphone optimisation), Medium ≤ 20 allocs.
**Q8d Lock:** Ratcliff-Obershelp Short ≤ 4 allocs (recursive decomposition allocates per recursion level; tighter budget judged too complex for marginal gain).
**Sources:** `docs/requirements.md §14.1, §14.2, §14.3`, §10 Tokenisation amendment.
**Reviewer reports closing:** algorithm-performance Important (×4).

### Gap 5 — `DefaultScorer` panics on internal inconsistency (Critical) — **RESOLVED**
**Finding (original):** `scorer.go:586–592` — `DefaultScorer()` panics if its internal construction fails with a bare untyped `panic("fuzzymatch: DefaultScorer construction failed (this is a bug): " + err.Error())`. This is intended as a "this is a bug" guard, but the panic value is an untyped string, so consumers cannot discriminate library-internal panics from caller-error panics via `recover()`.
**Resolution (LOCKED 2026-05-17):** **add a new sentinel `ErrInternalInvariantViolated`** and panic with that as a typed error wrapping the internal cause. Replace:
```go
panic("fuzzymatch: DefaultScorer construction failed (this is a bug): " + err.Error())
```
with:
```go
panic(fmt.Errorf("%w: DefaultScorer construction failed: %w", ErrInternalInvariantViolated, err))
```
**Semantic contract:**
- `ErrInternalInvariantViolated` NEVER fires in correct usage. Seeing it == library bug == file an issue.
- It MUST NOT be used to wrap caller-supplied parameter errors. Parameter errors use the dedicated parameter sentinels.
- It is the single typed-panic shape for "this should be impossible given the option-time validation and the immutable-after-construction Scorer contract."
- Consumers can `defer recover()` and call `errors.Is(panicValue.(error), ErrInternalInvariantViolated)` to discriminate.
- The four-section godoc template lives in `docs/requirements.md §6.A` per the standard `documentation-standards` skill.

**Companion test:** add `TestDefaultScorer_NeverPanics_PropertyTest` (or equivalent) asserting `DefaultScorer()` succeeds under all reasonable build configurations. The package-init-time assertion path (`var _ = func() bool { ... }()`) is rejected because it conflicts with the locked "no init-time side effects" position from the determinism reviewer (and is already flagged separately as an Improvement-sweep item Q14b).
