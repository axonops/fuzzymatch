# Phase 08.5 Plan 04: Q2 Data-vs-Parameter Validation Framework Guards Summary

Q2 strict-parameter guards now reject NaN, ±Inf, weight ≤ 0, α+β ≤ 0
at option-application time across three Scorer functional options;
the `TverskyScore` direct-call surface panics with typed-error values
(`fmt.Errorf("%w: …", sentinel, …)`) so `recover()` callers can
discriminate via `errors.Is`.

## What landed

Atomic single-commit landing of the Phase 8.5 Q2 data-vs-parameter
framework guards: three strict-parameter checks on the Scorer
functional options plus typed-error panic discipline on the direct
`TverskyScore` / `TverskyScoreRunes` surfaces. The plan's locked spec
(per `docs/requirements.md §6.A`) — comparison-data inputs lenient,
parameter inputs strict — is now enforced at every entry point that
admits a numeric parameter.

### scorer_options.go — three guards extended

1. **`WithThreshold(t)`** (lines 257-280 → 277-308 post-edit):
   - Added `math.IsNaN(t) || math.IsInf(t, 0)` BEFORE the existing
     `t < 0.0 || t > 1.0` range check.
   - NaN must be tested first because NaN compares false to both
     range tests; the prior check silently admitted NaN.
   - +Inf was already rejected (Inf > 1.0 is true) and -Inf was
     already rejected (-Inf < 0.0 is true); the explicit
     `math.IsInf` adds defence in depth.
   - Updated godoc to enumerate all rejection cases (NaN, ±Inf,
     t<0, t>1) and reference Phase 8.5 Q2.

2. **`WithAlgorithm(algo, weight)`** (lines 150-165 → 150-185
   post-edit):
   - Replaced `if weight <= 0` with the comprehensive guard
     `if math.IsNaN(weight) || math.IsInf(weight, 0) || weight <= 0`.
   - NaN must be tested first (same reason as WithThreshold);
     +Inf was previously admitted by `weight <= 0` (Inf > 0 is
     true), which would propagate as an unbounded value through
     the Scorer reduction.
   - Updated godoc to enumerate all rejection cases (NaN, ±Inf,
     weight≤0) and reference Phase 8.5 Q2.

3. **`WithTverskyAlgorithm(weight, alpha, beta, n)`** (lines 381-399
   → 411-475 post-edit):
   - Extended weight check to the same NaN/Inf/≤0 comprehensive
     guard pattern.
   - Added `math.IsNaN(alpha) || math.IsNaN(beta) || math.IsInf(alpha, 0) || math.IsInf(beta, 0)` check (returns ErrInvalidTverskyParam).
   - Preserved existing `alpha < 0 || beta < 0` check (returns
     ErrInvalidTverskyParam).
   - **Added the load-bearing `alpha + beta <= 0` guard** — closes
     the previously-documented panic-at-Score-time escape where
     `WithTverskyAlgorithm(_, 0, 0, _)` constructed successfully
     then panicked on the first `Score` call.
   - Updated godoc — the prior hedge ("the α + β > 0 constraint is
     enforced at runtime by TverskyScore itself; this option does
     not re-check it") is replaced with a clear enumeration of
     every rejection case + a Phase 8.5 Q2 reference.

### tversky.go — direct-call typed-error panic discipline

- Added `import "fmt"` and `import "math"`.
- Both `TverskyScore` and `TverskyScoreRunes` now panic with
  `fmt.Errorf("%w: …", sentinel, …)` values instead of bare strings.
  Four panic sites per file × two files = **4 typed-error panics in
  tversky.go**.
- Added NaN/Inf rejection on α and β at the direct-call surface
  (mirrors the option-time guard so both legs of the framework are
  consistent; the panic value wraps `ErrInvalidTverskyParam`).
- The wrapped error's `Error()` text still contains the original
  sentinel message ("fuzzymatch: invalid q-gram size" /
  "fuzzymatch: invalid tversky parameter") so log-scraping consumers
  are unaffected.
- Updated the file-level godoc block (the "Direct-call validation"
  section originally citing CONTEXT.md §5 LOCKED) to describe the
  Phase 8.5 Q2 supersession with typed-error panic values.

### tversky_test.go — typed-error panic-recovery assertions

- Updated `TestTversky_PanicsOnInvalidN`: assertion shape changed
  from `r.(string)` + `strings.Contains(msg, "invalid q-gram size")`
  to `r.(error)` + `errors.Is(err, ErrInvalidQGramSize)` +
  `strings.Contains(err.Error(), "invalid q-gram size")` (defence
  in depth). Byte and rune paths both updated.
- Updated `TestTversky_PanicsOnInvalidParams`: assertion shape
  upgraded to `errors.Is(_, ErrInvalidTverskyParam)`; **extended
  with seven new NaN/Inf rows** (alpha_nan, beta_nan, both_nan,
  alpha_pos_inf, alpha_neg_inf, beta_pos_inf, beta_neg_inf).
- New `TestTverskyScore_DirectCall` — named Q2 contract test for
  the data-vs-parameter framework. Four sub-cases (both_zero,
  alpha_nan, alpha_pos_inf, beta_neg_inf) each asserting
  `errors.Is(panicValue.(error), ErrInvalidTverskyParam)`.
