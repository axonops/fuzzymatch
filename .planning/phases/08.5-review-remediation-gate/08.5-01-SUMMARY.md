# Phase 8.5 Plan 01: Sentinel Hygiene Foundation Summary

Atomic single-commit landing of the v1.0 sentinel contract — renamed `ErrInvalidAlgorithm` → `ErrInvalidAlgoID` across 15 call sites in 5 files, added `ErrInvalidInnerAlgo` + `ErrInternalInvariantViolated`, removed three unused sentinels, rewrote every remaining sentinel's godoc into the 4-section template, and replaced two untyped-string panics with typed-error panics (`fmt.Errorf("%w: …", sentinel, …)`) discoverable via `errors.Is`.

## What Shipped

- **Sentinel rename (Gap 4):** `ErrInvalidAlgorithm` → `ErrInvalidAlgoID` atomically across `errors.go`, `errors_test.go`, `scorer.go`, `scorer_options.go`, `scorer_options_test.go`, `llms-full.txt`, plus the godoc references already on the post-rename name in `monge_elkan.go` (lines 162, 342) carry forward unchanged. `llms.txt`'s exported-symbol index also updated. Message text changes from `"fuzzymatch: invalid algorithm"` to `"fuzzymatch: invalid algorithm identifier"`.
- **New sentinel `ErrInvalidInnerAlgo`** (Q4 follow-up): inner-AlgoID rejection sentinel for Monge-Elkan. Message: `"fuzzymatch: invalid inner algorithm for Monge-Elkan composite"`.
- **New sentinel `ErrInternalInvariantViolated`** (Gap 5): typed-panic sentinel for library bugs. Message: `"fuzzymatch: internal invariant violated (library bug — please file an issue)"`.
- **Removed three unused sentinels (Q4):** `ErrInvalidInput`, `ErrInvalidConfiguration`, `ErrEmptyInput`. None had call sites in the repo.
- **Four-section godoc template applied** to `ErrInvalidQGramSize`, `ErrInvalidTverskyParam`, `ErrInvalidAlgoID` (post-rename), `ErrInvalidInnerAlgo` (new), `ErrInternalInvariantViolated` (new). The existing 4-section block on `ErrEmptyScorer` (lines 108-121, pre-Phase-8.5) was used as the reference template. The 4-section template is **What** / **Common causes** / **Resolution** / **Example**.
- **`DefaultScorer` typed panic (Gap 5):** `scorer.go` now imports `fmt` and the panic site (formerly `panic("fuzzymatch: DefaultScorer construction failed (this is a bug): " + err.Error())`) becomes `panic(fmt.Errorf("%w: DefaultScorer construction failed: %w", ErrInternalInvariantViolated, err))`. Consumers can discriminate via `recover()` + `errors.Is(panicValue.(error), ErrInternalInvariantViolated)`.
- **Monge-Elkan typed panic (Q4 follow-up):** `monge_elkan.go` now imports `fmt` and the direct-call panic at line 382 becomes `panic(fmt.Errorf("%w: AlgoID %s not permitted as Monge-Elkan inner metric", ErrInvalidInnerAlgo, inner.String()))`. The full error string is `"fuzzymatch: invalid inner algorithm for Monge-Elkan composite: AlgoID <Name> not permitted as Monge-Elkan inner metric"`. `MongeElkanScoreSymmetric` continues to delegate the panic to `MongeElkanScore`'s first call (no behavioural change to the symmetric variant's panic-surface contract).
- **Q1 spec catch-up:** the `errors.go` package-level header note that mentioned an unused `ErrUnequalLength` placeholder is removed. Hamming uses silent-max (no error sentinel), per docs/requirements.md §7.1.4 LOCKED.

### Four-section godoc template citation

The pre-existing `ErrEmptyScorer` declaration (now at lines ~155-167 in the updated `errors.go`; previously lines 108-121 per the plan's interface inventory) was used as the canonical four-section template:

```go
// ErrEmptyScorer indicates NewScorer was called without any algorithm
// option — the option slice contained zero WithAlgorithm /
// With*Algorithm entries by the time validation ran. A Scorer with no
// algorithms has no meaningful composite to compute.
//
// Pass at least one WithAlgorithm option (or use DefaultScorer() for
// the opinionated six-algorithm composition).
//
// Returned by NewScorer (Phase 8) after the missing-threshold check
// passes and the option-validation pipeline finds cfg.entries empty.
//
// Discriminate via errors.Is(err, fuzzymatch.ErrEmptyScorer); never
// match the error message string.
var ErrEmptyScorer = errors.New("fuzzymatch: ...")
```

`ErrInvalidQGramSize`, `ErrInvalidTverskyParam`, `ErrInvalidAlgoID`, `ErrInvalidInnerAlgo`, `ErrInternalInvariantViolated` all now follow the same **What** / **Common causes** / **Resolution** / **Example** rhythm.

Added `errors` to the test file's import list.
- **Files modified:** `monge_elkan_test.go`
