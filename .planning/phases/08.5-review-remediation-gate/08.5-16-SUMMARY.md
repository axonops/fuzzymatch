**release.yml needs:[ci] mechanism:** ci.yml is on `pull_request` + `push.branches: [main]` triggers, NOT on `push.tags`, so a plain in-workflow `needs: [ci]` cannot resolve cross-workflow on a tag push. Solution implemented in two layers:

1. A new `ci` job (named `verify CI green on tagged commit`) queries the GitHub Checks API via `gh api repos/{owner}/{repo}/commits/{sha}/check-runs` and asserts every `make check` check-run conclusion is `success`. Polls up to 10 minutes to handle the race between tag-push and the push-to-main CI run.
2. The `release` job declares `needs: [ci]` so it cannot start until the pre-flight passes.

**nightly.yml structure** (`.github/workflows/nightly.yml`, 268 lines):

1. **long-form-fuzz** — matrix over 36 root-package fuzzers; per-fuzzer `-fuzztime=${NIGHTLY_FUZZTIME}` (default 5m, overridable via workflow_dispatch input). Each runner snapshots `testdata/fuzz/<FuzzerName>/` into `fuzz-artefacts/${{ matrix.fuzzer }}` and uploads via `actions/upload-artifact@v7`.
2. **benchstat-regression** — runs on `${{ inputs.fallback_runner || '[self-hosted, fuzzymatch-bench]' }}`. The cron-driven default is the self-hosted shared runner (mask/audit hardware per STACK.md); the workflow_dispatch `fallback_runner` input lets a maintainer manually steer to `ubuntu-latest` when the self-hosted runner is unavailable, in which case `continue-on-error` is set so hardware noise does not block the nightly. Runs `make bench` + `make bench-compare` (the PERF-04 10% regression gate).
3. **auto-pr-fuzz-corpus** — gated on `needs: [long-form-fuzz]`. Downloads every fuzz-artefact, merges new `testdata/fuzz/<FuzzerName>/<hash>` entries into the working tree, and opens a PR via `peter-evans/create-pull-request@v7` with branch `auto/fuzz-corpus-${{ github.run_id }}`. The PR body documents the triage protocol; T-08.5-32 mitigation (controlled action + human merge approval; synthetic corpus only).
