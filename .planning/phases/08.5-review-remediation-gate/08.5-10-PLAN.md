<objective>
Q10 — add three new cross-validation corpora following the canonical pattern established by `scripts/gen-token-ratio-cross-validation.py`. Each corpus consists of:

1. A Python generator script (NOT a runtime dependency — `vectors.json` is committed).
2. A committed `vectors.json` file.
3. A Go-side loader test that consumes `vectors.json` and asserts fuzzymatch output equals (or is within tolerance of) the reference library output.
4. A Makefile regen target.
5. A CONTRIBUTING.md note about regen discipline.

Three corpora to add:
- **character/** — jellyfish==1.2.1 (Python — already pinned for phonetic) for Levenshtein, Jaro, JaroWinkler, DL-OSA.
- **qgram/** — py_stringmatching for Q-Gram Jaccard, Sørensen-Dice, Cosine, Tversky.
- **monge-elkan/** — py_stringmatching for Monge-Elkan with JaroWinkler-inner and Levenshtein-inner.

Pattern source: `scripts/gen-token-ratio-cross-validation.py` (RapidFuzz Indel formula) is the canonical template. Already-in-tree generators (`gen-phonetic`, `gen-ratcliff-obershelp`, `gen-swg`, `gen-token-ratio`) all share the same shape.

Output: 3 generators + 3 vectors.json files (~30-50 cases each) + 3 Go loader tests + 3 Makefile targets + 1 CONTRIBUTING note.

Existing generator template: scripts/gen-token-ratio-cross-validation.py

Generator canonical shape (PATTERNS.md §3 Shared — Cross-validation generator):
- File header: Apache 2.0 license + module docstring + reference-implementation citation + tokeniser-divergence handling note + version pin

- Output vectors.json with: meta block (version, generator script name, run date) + cases array (each case has all algorithm scores)
- Makefile regen target (`make regen-<corpus>-cross-validation`)
- CONTRIBUTING note about regeneration discipline

Go-side loader test template: token_ratio_cross_validation_test.go (TestTokenRatios_CrossValidation)

Reference-library version pins:
- jellyfish==1.2.1 (already used by gen-phonetic-cross-validation.py)

- Document the pin choice in the generator and in CONTRIBUTING.md

Determinism discipline:
- CASES are sorted, deterministic; no random sampling
- vectors.json is committed; CI reads from JSON, never invokes Python
- Python is required only for regeneration (e.g. on a reference-library version bump)
- Tolerance for float outputs: ≤ 1e-9 (or match what gen-token-ratio uses); for integer outputs (Levenshtein distance), exact equality
