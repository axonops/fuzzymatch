# Phase 08.5 Plan 13: Validate public surface (Q4 — the largest single feature in Phase 8.5) Summary

**One-liner:** Public `Validate(a, b string) []Warning` ships with `Warning` / `WarnKind` types, 5 WarnKind constants, an `AlgoIDAny` cross-cutting sentinel, a per-algorithm rule table, and the full unit + bench + fuzz + BDD test pyramid — closing VALIDATE-01..VALIDATE-05 and unblocking Plan 17's documentation cluster.

## What was delivered

### Public surface landed (5 new identifiers + 5 new constants + 1 sentinel)

### Benchmark alloc-counts achieved

```
goos: darwin / goarch: arm64 / cpu: Apple M2

BenchmarkValidate_NoWarnings_ASCII_Short-8     6_163_856    178.4 ns/op    394 B/op     5 allocs/op
BenchmarkValidate_WithWarnings_ASCII_Short-8   2_912_330    411.8 ns/op    651 B/op     8 allocs/op
BenchmarkValidate_PathologicallyLarge-8            1_401  854_043 ns/op  426_830 B/op  11 allocs/op
BenchmarkValidate_NonASCII_AllAlgos-8          1_824_206    656.9 ns/op    776 B/op    10 allocs/op
```

- **No-warnings fast path:** 178 ns / 5 allocs. The two Tokenise() calls dominate the alloc count (each contributes ~2 — slice header + scratch buffer per Phase 8.5 Q8b).
- **With-warnings typical case:** 412 ns / 8 allocs. 7 Warning values + 4 fmt.Sprintf detail strings + the backing-array growth.
- **Pathologically large (100 KiB per side):** 854 µs wall time — comfortably under the T-08.5-25 mitigation budget of "single-digit milliseconds". 11 allocs — input size dominates work, not allocations.
- **Non-ASCII (all 5 ASCII-only algos warn):** 657 ns / 10 allocs.

### Fuzz smoke

Properties enforced per-execution: never panics, every Warning.Kind in WarnKinds() range, every Warning.Algorithm in AlgoIDs()∪{AlgoIDAny}, every Detail valid UTF-8, output sorted by (Algorithm, Kind), two-call determinism.

## Threat Surface — Mitigations Verified

Per the plan's `<threat_model>`:

- **T-08.5-25 (DoS via pathologically large input):** Validate scans inputs in O(len(a) + len(b)) — the `Tokenise` calls inherit the Phase 8.5 Q8b ASCII fast path; no quadratic check anywhere. Measured: 200 KB inputs complete in 854 µs (well under "single-digit ms"). The `WarnPathologicallyLargeInput` rule itself is the mitigation hint to consumers — they gate at the call site rather than after-the-fact.
- **T-08.5-26 (Tampering via output ordering):** `sort.SliceStable` with the complete (Algorithm, Kind) sort key in validate.go guarantees deterministic ordering. Three regression gates: `TestValidate_DeterministicOrdering` (5 input shapes × byte-equality), the FuzzValidate determinism property (two-call check on every fuzz input), and the BDD scenario "Two calls return identical warnings".
