    ## Suppression composition
    Three numbered subsections:
    ### Rule 1 — per-item SilenceLint
    Item field; cheapest check; suppresses any pair involving the Item.
    ### Rule 2 — SuppressedPairs canonical pairs
    Config field. Pairs canonicalised once at Check entry via the Scorer's NormalisationOptions (D-05). Case-insensitive when the Scorer's normalisation downcases. Self-pairs (a == b post-normalisation) are silently kept and never emit (D-05).
    ### Rule 3 — cross-group identical-name default
    When CompareAcrossGroups=true and CompareIdenticalAcrossGroups=false (default), identical Names across different Groups are silently suppressed. Opt out by setting CompareIdenticalAcrossGroups=true.

    Composition: rules compose via OR — any rule firing suppresses. Applied pre-emission, never post.

    ## Threshold boost
    The cross-group effective threshold is min(1.0, scorer.Threshold() + cfg.CrossGroupThresholdBoost). The boost is a parameter (subject to strict Q2 validation per D-04: NaN, ±Inf, < 0, > 1 → ErrInvalidConfig). Zero-value boost = 0.0 (no boost). DefaultConfig bakes 0.05.
