- **DEVIATION LOCKED <date>:** TokenSetRatioScore returns `0.0` (not `1.0`) when EITHER Tokenise output is empty — bug-for-bug compat with RapidFuzz `token_set_ratio` (issue #110) which itself mirrors fuzzywuzzy. - **Three-way max LOCKED <date>:** the three branches are (a) Indel ratio of sorted-joined `diff_ab` vs sorted-joined `diff_ba` — the ONLY actual `indelRatio` call in TokenSetRatio; (b) analytical ratio of intersection-only-string vs intersection-plus-diff_ab-string computed from string LENGTHS only (no DP) — derives from RapidFuzz Pattern 5 `sect_ab_dist = (1 if sect_len != 0 else 0) + ab_len`, ratio `= 1 - dist / (sect_len + sect_ab_len)`; (c) analytical ratio for the diff_ba side. The maximum is taken via explicit `if` chain (NOT builtin `max`).
- **DET-03 sect_len LOCKED <date>:** intersection cardinality computed via integer-counter loop summing `len(k)` over sorted intersection keys (per RESEARCH.md Pitfall 5 — integer addition is associative, output is a scalar int, no string built from set iteration). - **Subset short-circuit LOCKED <date>:** when `intersection` is non-empty AND (`diff_ab` is empty OR `diff_ba` is empty) — i.e. one set is a subset of the other — return 1.0 directly without computing the three ratios (per RESEARCH.md Pattern 5 critical landmine 2). This is a separate gate from the both-empty / one-empty guards.

    `func TokenSetRatioScore(a, b string) float64` body structure (Claude's discretion for exact code per CONTEXT §6):
    1. `if a == b { return 1.0 }` (identity short-circuit BEFORE Tokenise)
    2. `tokensA := Tokenise(a, DefaultTokeniseOptions()); tokensB := Tokenise(b, DefaultTokeniseOptions())`
    3. `if len(tokensA) == 0 || len(tokensB) == 0 { return 0.0 }` (DEVIATION per Pitfall 2 LOCKED — applies to both-empty AND one-empty cases since the catalogue convention `both-empty → 1.0` is overridden here)
    4. Build sets via `setA := make(map[string]struct{}, len(tokensA)); for _, t := range tokensA { setA[t] = struct{}{} }` (and same for setB)
    5. Compute intersection key slice via sorted iteration: `intersectKeys := make([]string, 0, ...); for k := range setA { if _, ok := setB[k]; ok { intersectKeys = append(intersectKeys, k) } }; sort.Strings(intersectKeys)` — and similarly `diffABKeys`, `diffBAKeys`. Per RESEARCH.md Pitfall 5 — the OUTPUT consumed downstream is integer arithmetic over these slices (`sect_len = sum of len(k) + (len(intersectKeys)-1)` if non-empty), so the per-call sort guarantees deterministic ordering for any future debug-log / failure-message paths
    6. `if len(intersectKeys) > 0 && (len(diffABKeys) == 0 || len(diffBAKeys) == 0) { return 1.0 }` (subset short-circuit per RESEARCH.md Pattern 5 critical landmine 2)
    7. Compute integer-counter `sectLen` (sum of `len(k)` for k in sorted intersection + separator bytes); compute `abLen` (sum of `len(k)` for k in sorted diffABKeys + separators); compute `baLen` similarly. Then construct sorted-joined `diffABJoined := strings.Join(diffABKeys, " ")` and `diffBAJoined := strings.Join(diffBAKeys, " ")` (these joins are deterministic — fed from sorted slices, NOT from set iteration)
    8. Compute three ratios:
       - `r1 := indelRatio([]byte(diffABJoined), []byte(diffBAJoined))` (Indel of just-the-differences)
       - `r2 := analyticalRatio(sectLen, abLen)` where `analyticalRatio(sect, diff int) float64` returns `1.0 - float64(distHelper(sect, diff)) / float64(2*sect + spacerOrZero(sect) + diff)` — exact formula per RESEARCH.md Pattern 5: `sect_ab_dist = (1 if sect_len != 0 else 0) + ab_len`, `sect_ab_len = sect_len + (1 if sect_len != 0 else 0) + ab_len`, ratio `= 1 - dist / (sect_len + sect_ab_len)` — implementer derives the exact integer expression from the formula. Fresh transcription — no rapidfuzz code copied
       - `r3 := analyticalRatio(sectLen, baLen)`
    9. Three-way max via explicit if-chain: `best := r1; if r2 > best { best = r2 }; if r3 > best { best = r3 }; return best`
