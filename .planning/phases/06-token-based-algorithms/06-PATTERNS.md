### `token_indel.go` (utility, transform)

**Analog:** `q_gram.go`

**File-header pattern** (`q_gram.go:1-87`): Apache-2.0 header (lines 1-13) → `Source:` block citing primary academic source (lines 22-25) → `Algorithm — <name>:` block describing the function contract (lines 27-42) → `Determinism (DET-03):` block (lines 43-52) → `Source-origin discipline:` block (lines 54-63) → `Implementation discipline:` block (lines 65-77) → `Allocation budget:` block (lines 79-86) → `package fuzzymatch` declaration. Every Phase 6 algorithm `.go` file follows this header layout (RESEARCH.md "Established Patterns: Source-Origin Statement format").

**Foundation kernel godoc + signature pattern** (`q_gram.go:90-117` for `extractQGrams`):
```go
// extractQGrams returns the multiset of overlapping length-n BYTE
// q-grams of s. Returns a non-nil empty map[string]int{} when n < 1,
// when s is empty, or when len(s) < n.
//
// Multiset semantics: ...
//
// The returned map MUST NOT be iterated by callers on any output path
// (DET-03). ...
func extractQGrams(s string, n int) map[string]int {
    if n < 1 || len(s) < n {
        return map[string]int{}
    }
    m := make(map[string]int, len(s)-n+1)
    for i := 0; i <= len(s)-n; i++ {
        m[s[i:i+n]]++
    }
    return m
}
```

**Apply to** `token_indel.go`: declare `lcsLen(a, b []byte) int`, `indelRatio(a, b []byte) float64`, `lcsLenRunes(a, b []rune) int`, `indelRatioRunes(a, b []rune) float64`. Each gets the Phase 5 godoc layout: contract paragraph + edge-case enumeration (both-empty / one-empty / identity) + DET-03 / DET-06 reaffirmation + alloc budget per RESEARCH.md §4.1.

**Two-row DP buffer pattern (stack fast path):** see `lcsstr.go:142-150`:
```go
m, n := len(a), len(b)
if n <= maxStackInputLen && isASCII(a) && isASCII(b) {
    var buf [(maxStackInputLen + 1) * 2]int
    maxLen, endI := lcsstrDP(a, b, m, n, buf[:n+1], buf[n+1:2*(n+1)])
    ...
}
maxLen, endI := lcsstrDP(a, b, m, n, make([]int, n+1), make([]int, n+1))
```

**Apply to** `lcsLen`: same pattern but the DP recurrence is the LCS-subsequence variant (Wagner-Fischer 1974), NOT the substring variant. The Phase 6 RESEARCH.md Pitfall 6 mandates an inline godoc paragraph contrasting `lcsLen` with `lcsstr.go`'s `LongestCommonSubstring` to prevent future contributors from confusing them. The `maxStackInputLen` constant is declared in `levenshtein.go` and referenced (do NOT redeclare). The `isASCII` helper is in `normalise.go` and referenced.

**Explicit `if/else` instead of builtin `min`/`max`** (`qgram_jaccard.go:240-246`):
```go
// min(cs, cl) — Go 1.26 stdlib has builtin min, but the
// project canonical pattern is the explicit if/else for
// determinism-reviewer auditability.
if cs < cl {
    intersection += cs
} else {
    intersection += cl
}
```

**Apply to** `lcsLen`'s `max(prev[j], curr[j-1])`: write as explicit `if prev[j] > curr[j-1] { curr[j] = prev[j] } else { curr[j] = curr[j-1] }` — load-bearing for determinism-reviewer auditability.

### `token_sort_ratio.go` / `token_set_ratio.go` / `partial_ratio.go` / `token_jaccard.go` (algorithms)

**Analog:** `qgram_jaccard.go` (file structure, identity short-circuit pattern, helper-function decomposition for kernel computation, Apache-2.0 + Sources block)

**Identity-short-circuit-then-empty-guard pattern** (`qgram_jaccard.go:144-156`):
```go
func QGramJaccardScore(a, b string, n int) float64 {
    if n < 1 {
        panic("fuzzymatch: invalid q-gram size")
    }
    if a == b {
        return 1.0 // identity short-circuit (covers both-empty too)
    }
    if a == "" || b == "" {
        return 0.0
    }
    qa := extractQGrams(a, n)
    qb := extractQGrams(b, n)
    return jaccardFromQGramMaps(qa, qb)
}
```

**Apply to** all four token-tier algorithms: identity short-circuit BEFORE any tokenisation (avoids `Tokenise` allocation on identical inputs). Empty-guard AFTER identity. Then call `Tokenise(s, DefaultTokeniseOptions())` per RESEARCH.md "Reusable Assets" (DefaultTokeniseOptions has Lowercase: true).

**Token Jaccard set-construction pattern (RESEARCH.md §8 Pattern 8):** see `qgram_jaccard.go:213-260` `jaccardFromQGramMaps` for the integer-counter pattern that satisfies DET-03 (the OUTPUT is an integer count, NOT an ordered slice). Token Jaccard differs from Q-Gram Jaccard ONLY in that tokens are SET-deduplicated (not multiset-counted) — use `map[string]struct{}` instead of `map[string]int`. RESEARCH.md "Pattern 8" example body (lines 454-491) is the verbatim implementation template.

```go
// Edge case (deviation from the catalogue convention):
```
