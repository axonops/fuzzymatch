<recorded_resolutions>
Plan SUMMARY.md must record:
- **Set vs Multiset LOCKED <date>:** TokenJaccard uses SET semantics (deduplicated tokens — `map[string]struct{}` not `map[string]int`), DISTINCT from Phase 5's Q-Gram Jaccard which uses MULTISET semantics on q-grams. Documented in algorithm godoc with the RESEARCH.md Pattern 8 distinction note. The semantic divergence is intentional: token presence is a binary signal in this context (a token appearing twice in the input doesn't make it "more present"); q-gram presence is a multiplicity signal (longer overlapping runs of identical q-grams indicate stronger similarity).
- **DET-03 intersection-cardinality LOCKED <date>:** Compute `|A ∩ B|` via integer-counter loop over the SMALLER set — `for k := range small { if _, ok := large[k]; ok { intersection++ } }` — output is a scalar `int` (associative integer addition; map iteration order does not affect the output). Then `union := len(setA) + len(setB) - intersection`. NO slice ever built from set iteration on the output path. DET-03 satisfied without any sort.
- **Both-empty STANDARD convention LOCKED <date>:** TokenJaccardScore("","") returns 1.0 (catalogue standard both-empty identity convention) — DOES NOT deviate like TokenSetRatio (which returns 0.0 per RapidFuzz issue #110). Property test guards `if x == "" { return true }` only on the IDENTITY (since identity holds vacuously here too) — but not on a deviation rationale.
- **Hand-derived reference vectors LOCKED <date>:** 4-6 hand-derived `(a, b)` pairs per CONTEXT §1b LOCKED: identical token sets → 1.0; disjoint token sets → 0.0; partial overlap (`"a b c"` / `"b c d"` → 2/4 = 0.5); subset (`"a b"` / `"a b c"` → 2/3 ≈ 0.6667); Unicode tokens (lowercase ASCII at corpus level — Tokenise lowercases). Each test case has the formula derivation embedded in the test comment per algorithm-correctness-standards "Reference Vectors".
